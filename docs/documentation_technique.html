Présentation du projet

HealthAI MSPR est une application web full-stack de suivi et d'analyse de données de santé, conçue pour FIREX GROUP. Elle permet la collecte, la visualisation et l'analyse des données physiologiques des utilisateurs (activité physique, alimentation, métriques santé, objectifs).

Objectifs

  • ✓Centraliser les données de santé des utilisateurs
  • ✓Offrir un tableau de bord d'analyse pour les administrateurs
  • ✓Permettre aux utilisateurs de consulter leur profil santé
  • ✓Gérer les flux de données et leur validation
  • ✓Détecter et nettoyer les données incohérentes
  • ✓Automatiser les pipelines de données avec Airflow
  • ✓Sécuriser l'accès via authentification JWT
  • ✓Déploiement 100% conteneurisé avec Docker

Architecture technique

L'application repose sur une architecture 3-tiers conteneurisée : présentation (Vue.js + Nginx), logique métier (FastAPI), persistance (PostgreSQL). Un pipeline Airflow orchestre les tâches de données en arrière-plan.

🌐 Navigateur (Chrome / Firefox)
↕ HTTP/HTTPS (port 89)
🖥️ Frontend — Vue 3 + Nginx
↕ REST API (proxy interne port 8000)
⚙️ Backend — FastAPI (Python)
🔐 Auth — JWT / bcrypt
↕ SQLAlchemy ORM
🗄️ PostgreSQL — Base de données santé
↕ Accès DB (port 5432 interne)
🌀 Apache Airflow — Orchestration DAGs
🗄️ PostgreSQL — Base Airflow

→
6. Bearer token
dans chaque requête

Structure du token JWT

Header

{ "alg": "HS256", "typ": "JWT" }

Payload

{ "sub": "1001", // ID utilisateur "is_admin": false, // Rôle "exp": 1741945200 // +8 heures }

Signature

HMACSHA256( base64(header) + base64(payload), SECRET_KEY // .env )

Paramètres de sécurité

ParamètreValeurRôle
ALGORITHMHS256Algorithme de signature symétrique
ACCESS_TOKEN_EXPIRE_MINUTES480 (8h)Durée de validité du token
SECRET_KEYVariable d'environnementClé secrète de signature (non exposée)
pwd_contextbcrypt (rounds auto)Hash adaptatif des mots de passe

Gestion côté frontend

  • Le token est décodé côté client pour extraire sub (user ID) et is_admin
  • isAuthenticated() vérifie à la fois la présence du token ET son expiration (champ exp)
  • Toute réponse 401 provoque un logout automatique et une redirection vers /connexion
  • Le token est stocké dans localStorage sous la clé app_token
  • Les guards de navigation Vue Router bloquent l'accès aux routes protégées si non authentifié

Nettoyage automatique des données (page Admin)

La page Nettoyage analyse en temps réel les données via l'API et signale :

  • Objectifs avec statut refusé ou rejeté
  • Objectifs sans description
  • Consommations avec calories ou quantité invalides (≤ 0)
  • Métriques avec fréquence cardiaque incohérente (< 30 ou > 220 bpm)
  • Métriques avec poids incohérent (< 20 ou > 350 kg)

Sécurité

Authentification & Autorisation

  • JWT HS256 — Tokens signés, expiration 8h
  • bcrypt — Hash adaptatif des mots de passe, jamais stockés en clair
  • OAuth2 Password Flow — Standard RFC 6749
  • RBAC — 2 rôles : user et admin
  • Vérification du token à chaque endpoint protégé
  • Déconnexion automatique si token expiré (frontend)

Protection des endpoints

  • Routes publiques : /login, /utilisateurs/register
  • Routes auth : lecture des données (GET)
  • Routes admin-only : suppressions (DELETE) et modifications globales
  • Validation stricte avec Pydantic v2 sur toutes les entrées
  • Path parameters validés (gt=0 pour les IDs)

Infrastructure

  • Ports DB exposés uniquement sur 127.0.0.1
  • SECRET_KEY via variable d'environnement (jamais en dur)
  • Réseau Docker interne isolé (health_net)
  • CORS configuré (à restreindre en production)

Frontend

  • Guards de navigation empêchent l'accès non autorisé
  • Vérification expiration JWT côté client (champ exp)
  • Gestion des 401 : logout + redirection automatique
  • Pas de données sensibles exposées dans le DOM

Limites et améliorations recommandées

Risque actuelRecommandation
Token stocké en localStorage (vulnérable XSS)Migrer vers httpOnly cookies en production
CORS allow_origins=["*"]Restreindre à l'origine frontend réelle
Pas de refresh tokenImplémenter un mécanisme de renouvellement silencieux
Pas de rate limiting sur /loginAjouter une limite de tentatives (brute force)
